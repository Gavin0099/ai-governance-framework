# Gotchas

- A PR summary is for reviewers, not for congratulating the author.
- Do not claim the change is safe, complete, or verified beyond actual evidence.
- If only focused tests or smoke ran, say that explicitly instead of implying full-suite coverage.
- Keep scope boundaries visible. Reviewers should not have to infer what was intentionally left untouched.
- When trust, release, and review surfaces disagree, name the disagreement instead of smoothing it into generic confidence language.
