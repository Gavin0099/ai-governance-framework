# Gotchas

- This skill is deprecated as a standalone PR handoff surface. Route to `reviewer-handoff`.
- Keep `pr_handoff` terminology for workflow artifacts, but do not create a separate PR-summary doctrine here.
- Do not claim the change is safe, complete, or verified beyond actual evidence.
- If only focused tests or smoke ran, say that explicitly instead of implying full-suite coverage.
- When trust, release, and review surfaces disagree, name the disagreement in the reviewer-handoff packet.
