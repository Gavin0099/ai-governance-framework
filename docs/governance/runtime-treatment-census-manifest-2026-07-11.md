# Runtime Treatment Census — Pinned Manifest and Evaluation (2026-07-11)

Status: reproducible census manifest with reviewer-judged evaluation columns.
No surface is retired, merged, downgraded, or modified by this document.

## Why this exists

An earlier in-chat classification of runtime treatment surfaces could not be
independently reproduced because its counting units were never pinned. This
manifest fixes that: the unit list is generated by pinned commands, each unit
carries mechanical fields anyone can recompute, and judged fields are labeled
as reviewer judgment with reasons.

## Pinned unit generation (re-runnable)

Units are exactly the union of these tracked-file sets at the commit that
introduces this manifest:

```text
git ls-files "runtime_hooks/*.py"
git ls-files "runtime_hooks/core/*.py"
git ls-files "runtime_hooks/adapters/*.py"
git ls-files "runtime_hooks/adapters/*/*.py"
git ls-files "templates/*.yaml"
plus two named prompt carriers:
  governance/copilot-instructions-template.md
  examples/multi-validator-contract/AGENTS.md
```

**N = 33.** Mechanical fields: `loc` = line count; `tests` = count of
`tests/test_*.py` whose filename contains the unit's base name (a containment
match; shared test files count for every unit they name); `last` = last commit
date touching the unit.

## Evaluation columns

- `class`: behavior_change | audit | mixed | carrier | consumer_pending —
  reviewer judgment of the unit's primary justification.
- `decision_evidence`: measured | observed | none — whether any decision
  effect of this unit has ever been measured or directly observed.
- `dup`: named duplication/overlap, if any.
- `note`: judgment reason, comprehension gloss state, or focused-review flag.

## Census findings (judgment, not disposition)

1. **behavior_change is small and concentrated**: 2 carrier documents
   (copilot template, consumer AGENTS.md exemplar) plus the
   `behavioral_advisory` signal families inside the two `*_task_check`
   modules. The v3 recalibration raises their burden of proof, owner-adopted
   and task-class-scoped only.
2. **decision_evidence is `none` for most units**: only the closeout chain
   (session_end, _canonical_closeout) has directly observed downstream
   consumption (F-7 / P1-C evidence). No unit here has a measured decision
   effect; the only measured precedent in the repo remains codex-review-fast.
3. **Focused-review flags** (candidates for their own read-only slices, in
   value order): gemini adapters (no known consumer), payload_audit_logger
   (log consumer unverified), decision_policy_v1_runtime (595 loc, zero
   containment-matched tests).
4. **Duplication is structural, not accidental**: the 4-platform adapter
   pattern is intentional; the copilot-template / AGENTS.md overlap is the
   one real content-duplication candidate.
5. **Human comprehensibility**: reviewer-facing outputs route through
   human_summary and the maturity/F-7 summaries; per the failure-driven rule,
   no new readability surface is proposed here.
