# 為什麼 AI Agent 需要治理？

> **核心主軸**：AI agent 的問題不是不夠聰明，而是缺少工程流程裡的證據鏈。

---

## 1. 開場：AI agent 已經開始改 code，但我們真的知道它改對了嗎？

現在的 AI 已經不只是聊天工具。

它開始能幫工程師寫 code、修改文件、產生測試、分析錯誤，甚至透過 agent workflow 連續完成多個步驟。

但這帶來了一個新問題：

> AI 產出的內容看起來可能很完整、很合理，甚至很有信心。
> **但「看起來合理」不等於「真的正確」，「AI 說完成」也不等於「工程上可以接受」。**

---

## 2. LLM 的本質限制

LLM 根據訓練資料與當下上下文，產生最可能符合需求的輸出。它很擅長補全、推理與生成，但它不天然具備工程流程裡需要的幾件事：

- 明確的**責任邊界**
- 可驗證的**完成證據**
- 有意義的**變更追蹤**
- 主動的**風險標示**

此外，RLHF 等偏好訓練讓模型傾向提供有幫助、完整、看似有信心的回應。在不確定時，它不會自然停下來說「我不知道」，而是產生一個流暢但可能低可信度的答案。

**重點不是批評 LLM，而是建立一個前提：**

> AI 很會產生結果，但結果不會自動變得可信。

---

## 3. 從 Prompt 到 Skill 到 Agent 的演化

| 階段 | 核心能力 | 問題 |
|---|---|---|
| **Prompt** | 用自然語言要求 AI 做事 | 結果依賴上下文與表達品質 |
| **Skill** | 把常用流程封裝成可重複能力 | 能力變強，但仍缺少治理邊界 |
| **Agent / Hermes** | 跨步驟執行、修改、驗證、回報 | 影響範圍變大，錯誤成本也變大 |
| **AI Governance** | 對行為建立邊界、證據、驗證與責任鏈 | — |

不是因為 AI 不好，所以要 governance；而是**因為 AI 變強了，所以需要 governance**。

### 為什麼 Agent 特別危險？

單次 prompt 答錯，頂多浪費一次時間，重來就好。

但 agent 連續執行 5 個步驟後答錯，前面 4 步的副作用可能已經無法回滾：檔案已修改、PR 已開出、測試已跑過、狀態已變更。

**錯誤會被放大，而不是被隔離。**

---

## 4. AI Agent 帶來的工程風險

| 風險 | 描述 |
|---|---|
| **改太多** | 沒有 scope 限制，AI 可能改到不該改的地方 |
| **沒有證據** | AI 宣稱完成，但沒有 artifact 支撐 |
| **測試不足** | 缺少 smoke / unit / integration 驗證 |
| **難以 review** | 改動範圍大、脈絡不清，reviewer 難以判斷 |
| **難以追責** | 無法事後還原「是誰改的、為什麼改」 |
| **難以 rollback** | 多步驟交錯，無法乾淨還原到某個狀態 |

---

## 5. AI Governance 到底管什麼？

五個問題：

1. **Scope**：這次 AI 被允許改什麼？不允許改什麼？
2. **Evidence**：AI 宣稱完成，有沒有 artifact 作為證據？
3. **Validation**：有沒有 smoke test / unit test / 結構驗證支撐？
4. **Traceability**：之後誰能知道 AI 改了什麼、為什麼改？
5. **Human Review**：最後的判斷權，是否仍然保留在人類工程流程裡？

---

## 6. 最小可行流程

```
Task → AI Change → Receipt → Smoke → Risk Report → Human Review
```

每一步都有對應的問題：

| 步驟 | 問題 |
|---|---|
| Task | 這次任務的邊界在哪？ |
| AI Change | AI 實際改了什麼？ |
| Receipt | 有沒有 closeout 記錄？ |
| Smoke | 基本功能還正常嗎？ |
| Risk Report | 這次改動的風險等級是什麼？ |
| Human Review | 人類最後確認後才合併 |

---

## 7. 結論

AI governance 不是要讓 AI 變慢，而是讓 AI 的速度可以被工程流程承接。

它讓 AI agent 的輸出變得：

- **可 review**：有脈絡、有 scope、有 artifact
- **可驗證**：有 smoke、有測試結果
- **可追蹤**：有 session record、有 commit linkage
- **可回滾**：有邊界限制，副作用可控
- **可接受**：進得了真正的工程 PR 流程

> AI 不只是寫得快，而是能安全地進入真正的工程流程。
