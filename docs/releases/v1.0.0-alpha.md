# v1.0.0-alpha

發版日期：2026-03-15

`v1.0.0-alpha` 的歷史意義是：這個 repo 第一次不再只是 governance 文件集合，而是變成可執行的 AI coding runtime-governance framework prototype。

## 這個 alpha 當時納入了什麼

### Runtime Governance Spine

- `session_start`
- `pre_task_check`
- `post_task_check`
- `session_end`
- memory curation / promotion pipeline

### Built-In Rule Packs

- `common`
- `refactor`
- `python`
- `cpp`
- `csharp`
- `swift`
- `avalonia`
- `kernel-driver`

### Supported AI Adapter Surfaces

- Claude Code
- Codex
- Gemini

### Review / Audit Outputs

- `change_control_summary.py`
- `change_control_index.py`
- `session_end` summary artifacts
- curated memory artifacts

### External Domain Contract Seam

當時已驗過的 example 與 external repo 包括：

- `examples/usb-hub-contract`
- `USB-Hub-Firmware-Architecture-Contract`
- `Kernel-Driver-Contract`

當時的 practical status 大致是：

- USB-Hub firmware：第一個真實 external vertical slice
- Kernel driver：第二個 low-level domain slice，已有 multi-validator advisory post-task loop
- runtime failure handling：第一個 execution-layer failure slice 已存在

### Adoption / Onboarding Tooling

- `requirements.txt`
- `start_session.md`
- `quickstart_smoke.py`
- `example_readiness.py`
- `docs/releases/alpha-checklist.md`
- CI-generated trust-signal snapshot bundle
- stable repo-local generated status path：`docs/status/generated/`
- stable repo-local generated release-package path：`docs/releases/generated/`
- external repo hook install / verify / readiness / smoke / onboarding report / onboarding index

## 這個 alpha 適合拿來做什麼

- 想要 AI coding governance，但不想只停在 prompt-only interaction 的團隊
- 想要 reviewable AI session artifact 的 repo
- 想探索 firmware、driver、或其他 high-context codebase governance 的早期採用者

## 當時的已知邊界

這個 alpha 當時仍刻意保守，不提供：

- fully closed workflow interception
- 對多數 domain validator 的 hard-stop policy enforcement
- 接近 true policy engine 的 deep semantic reasoning
- full plugin marketplace 或 versioned plugin API surface
- 在不穩定 runtime 條件下的完整 async / re-evaluation determinism harness
