# v1.0.0-alpha

發版日期：2026-03-15

`v1.0.0-alpha` 是這個 repo 第一次把 runtime governance spine、built-in rule packs、review output、external domain seam 與 adoption tooling 一起收成可對外描述的 alpha 版本。

## 這版 alpha 納入了什麼

### Runtime Governance Spine

- `session_start`
- `pre_task_check`
- `post_task_check`
- `session_end`
- memory curation / promotion pipeline

### Built-In Rule Packs

- `common`
- `refactor`
- `python`
- `cpp`
- `csharp`
- `swift`
- `avalonia`
- `kernel-driver`

### Supported AI Adapter Surfaces

- Claude Code
- Codex
- Gemini

### Review / Audit Outputs

- `change_control_summary.py`
- `change_control_index.py`
- `session_end` summary artifacts
- curated memory artifacts

### External Domain Contract Seam

這版已包含 example 與 external repo seam：
- `examples/usb-hub-contract`
- `USB-Hub-Firmware-Architecture-Contract`
- `Kernel-Driver-Contract`

### Adoption / Onboarding Tooling

- `requirements.txt`
- `start_session.md`
- `quickstart_smoke.py`
- `example_readiness.py`
- `docs/releases/alpha-checklist.md`
- CI-generated trust-signal snapshot bundle
- stable repo-local generated status path：`docs/status/generated/`
- stable repo-local generated release-package path：`docs/releases/generated/`
- external repo hook install / verify / readiness / smoke / onboarding report / onboarding index

## 這版 alpha 的目的

- 建立 AI coding governance 的最小可執行 runtime spine
- 建立可 review 的 AI session artifact 與 repo-facing evidence path
- 驗證 firmware、driver 等高上下文 codebase 也能被帶入 governance workflow

## 這版 alpha 不主張什麼

- fully closed workflow interception
- 每個 domain validator 都已成為 hard-stop policy enforcement
- true policy engine 等級的 deep semantic reasoning
- full plugin marketplace 或 versioned plugin API surface
- 完整 async / re-evaluation determinism harness

1. `pip install -r requirements.txt`
2. `python governance_tools/quickstart_smoke.py --project-root . --plan PLAN.md --contract examples/usb-hub-contract/contract.yaml --format human`

4. `python governance_tools/trust_signal_snapshot.py --project-root . --plan PLAN.md --release-version v1.0.0-alpha --contract examples/usb-hub-contract/contract.yaml --publish-docs-status --format human`

## Alpha 定位

這個 alpha 的作用是把：
- runtime loop
- review surface
- release / adoption signal
- external contract seam

先收成同一條 bounded 主線。

## Known Limits

這個 release 仍是 bounded prototype，不應被描述成 fully closed policy engine。

- workflow interception coverage 仍為 partial
- 多數 domain enforcement 仍為 mixed / advisory-first
- semantic verification 仍淺於 full policy engine
- external contract seam 雖然存在，但仍是 versioned contract / plugin marketplace 的前置形態
