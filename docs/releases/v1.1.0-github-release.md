# v1.1.0 GitHub Release 草稿

`ai-governance-framework` `v1.1.0` 是把 token-optimization roadmap、external adoption path 與 baseline governance surface 收斂成第一個可正式對外描述的 beta-ready release。

說明：
- 這份文件用作 `v1.1.0` 的 GitHub Release body draft。
- 正式事實請以 [v1.1.0.md](v1.1.0.md) 與 [CHANGELOG.md](../../CHANGELOG.md) 為準。
- `main` 分支後續較新的 closeout / advisory / runtime hardening 不應回寫成 `v1.1.0` 已發版能力。

## 亮點

- **跨平台 adoption path**：`adopt_governance.py` 取代 bash-only 初始化路徑，成為 canonical adoption entrypoint。
- **Beta gate 達成**：external repo adoption path 已可被測試、檢查與重現。
- **Token optimization 完成**：完成 token payload 收斂與高頻 surface 壓縮。
- **Output-tier separation**：建立 Tier 1 / 2 / 3 output 分層。
- **1,386 tests**：相較 `v1.0.0-alpha` 的 835 tests 顯著成長。

## 本版納入的重點

### Adoption Toolchain

- `adopt_governance.py`：Windows-safe、cross-platform adoption entrypoint
- `--refresh`、`--dry-run`
- framework root auto-discovery
- 16-check drift checker
- `minimum-legal-schema.md` 作為 onboarding reference

### Runtime Infrastructure

- task-level detector：`L0` / `L1` / `L2`
- domain summary loader 與 summary-first contract loading
- payload audit logger
- Windows-safe terminal output fallback

### Governance Documents / Authority Layer

- `governance/AUTHORITY.md`
- `governance/RULE_REGISTRY.md`
- `governance_tools/authority_loader.py`
- `governance_tools/rule_classifier.py`
- `governance_tools/task_level_detector.py`
- `governance_tools/output_tier.py`

## 建議驗證步驟

1. `pip install -r requirements.txt`
2. `python governance_tools/quickstart_smoke.py --project-root . --plan PLAN.md --contract examples/usb-hub-contract/contract.yaml --format human`
3. `python governance_tools/trust_signal_overview.py --project-root . --plan PLAN.md --release-version v1.1.0 --contract examples/usb-hub-contract/contract.yaml --format human`
4. `python governance_tools/trust_signal_snapshot.py --project-root . --plan PLAN.md --release-version v1.1.0 --contract examples/usb-hub-contract/contract.yaml --publish-docs-status --format human`
5. `python governance_tools/trust_signal_publication_reader.py --project-root . --docs-status --format human`
6. `python governance_tools/release_package_snapshot.py --version v1.1.0 --publish-docs-release --format human`
7. `python governance_tools/release_package_reader.py --version v1.1.0 --project-root . --docs-release --format human`
8. `python governance_tools/release_package_publication_reader.py --project-root . --docs-release-root --format human`

10. `bash scripts/verify_phase_gates.sh`

## 必須同時揭露的限制

這個 release 仍是 bounded prototype，不應被描述成 fully closed policy engine。

- workflow interception coverage 仍為 partial
- domain enforcement 仍為 advisory-first / mixed
- rule classification 仍依 repo type 與 rule pack 推定
- semantic verification 仍未達 full policy-engine depth
