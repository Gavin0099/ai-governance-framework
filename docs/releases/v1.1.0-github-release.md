# v1.1.0

`ai-governance-framework` v1.1.0 completes the token-optimization roadmap and delivers a fully validated external adoption path, advancing the framework to Beta.

## Highlights

- **Cross-platform adoption** — `adopt_governance.py` is now the canonical adoption entrypoint, with full Windows support replacing the bash-only `init-governance.sh`
- **Beta Gate reached** — first real external repo (`Hearth`, domain: `household-finance`) completed a full `session_start → pre_task → post_task` lifecycle across all three harnesses (Claude Code, Codex, Gemini) without author intervention
- **Token optimization completed** — Steps 1–7 roadmap finished; combined payload reduced from ~104,696 to ~49,202 tokens (−53%)
- **Output-tier separation** — Tier 1/2/3 output design reduces cost on UI-level and onboarding-shaped tasks significantly
- **1,386 tests** — up from 835 at v1.0.0-alpha

## New in This Release

### Adoption Toolchain

- `adopt_governance.py` — Windows-safe, cross-platform adoption with `--refresh`, `--dry-run`, and framework root auto-discovery
- 16-check drift checker including placeholder detection and `AGENTS.md` fill validation
- `minimum-legal-schema.md` — onboarding reference that removes the need to read source code

### Runtime Infrastructure

- Task-level detector (`L0` / `L1` / `L2`) with domain gate short-circuit
- Domain summary loader with summary-first contract loading
- Payload audit logger with distinct onboarding lane
- Windows-safe terminal output fallback

### Governance Documents

- `governance/AUTHORITY.md` — authority table
- `governance/RULE_REGISTRY.md` — 14 rule packs with classification
- `governance_tools/authority_loader.py`, `rule_classifier.py`, `task_level_detector.py`, `output_tier.py`

## Known Limits

This is still a prototype framework, not a fully closed policy engine:

- Workflow interception coverage is partial
- Most domain enforcement remains advisory-first
- Rule classification is not yet differentiated by repo type
- Semantic verification depth is improving but not at full policy-engine level
