# v1.1.0

`ai-governance-framework` v1.1.0 完成當時的 token-optimization roadmap，並交付可驗證的 external adoption path，讓 framework 從 alpha 原型往 Beta 前進一步。

注意：

- 這份文件是 GitHub release body draft
- 它代表的是 `v1.1.0` 發版當時的公開敘事
- 不等於今天 `main` 分支的全部能力

## Highlights

- **Cross-platform adoption**：`adopt_governance.py` 成為 canonical adoption entrypoint，Windows adoption 正式成立，取代 bash-only 的 `init-governance.sh`
- **Beta Gate reached**：第一個真實 external repo adoption path 被完整驗證
- **Token optimization completed**：完成 Steps 1–7，payload 明顯下降
- **Output-tier separation**：Tier 1 / 2 / 3 output 設計落地
- **1,386 tests**：相較 `v1.0.0-alpha` 的 835 tests 明顯成長

## This Release 主要新增

### Adoption Toolchain

- `adopt_governance.py`：Windows-safe、cross-platform adoption entrypoint
- `--refresh`、`--dry-run`、framework root auto-discovery
- 16-check drift checker
- `minimum-legal-schema.md` 作為 onboarding reference

### Runtime Infrastructure

- task-level detector（`L0` / `L1` / `L2`）
- domain summary loader 與 summary-first contract loading
- payload audit logger
- Windows-safe terminal output fallback

### Governance Documents / Authority Layer

- `governance/AUTHORITY.md`
- `governance/RULE_REGISTRY.md`
- `governance_tools/authority_loader.py`
- `governance_tools/rule_classifier.py`
- `governance_tools/task_level_detector.py`
- `governance_tools/output_tier.py`

## 當時已知邊界

這版 release 仍是 bounded prototype，不是 fully closed policy engine：

- workflow interception coverage 仍然 partial
- 多數 domain enforcement 仍是 advisory-first / mixed
- rule classification 尚未按 repo type 完整分化
- semantic verification 仍淺於 full policy-engine 層級
