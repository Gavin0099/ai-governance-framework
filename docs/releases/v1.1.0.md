# v1.1.0

Release date: 2026-03-22

This release advances the framework from alpha prototype to Beta, completing the token-optimization roadmap (Steps 1–7) and delivering a fully validated external adoption path.

## What Changed in v1.1.0

### Cross-Platform Adoption Toolchain

- `adopt_governance.py` replaces `bash scripts/init-governance.sh` as the canonical adoption entrypoint, with full Windows support
- `--refresh` mode re-hashes existing baselines and updates section inventory
- `--dry-run` flag previews changes without writing
- Framework root auto-discovery via `GOVERNANCE_FRAMEWORK_ROOT` env or upward scan

### Governance Drift and Schema Visibility

- 16-check drift checker with placeholder detection, template-copy guard, inventory staleness, and `AGENTS.md` fill checks
- `adopt` substitutes repo slug and today's date into templates, repairs the required `AGENTS.base.md` seam
- `minimum-legal-schema.md` surfaced at all onboarding touch points
- `AGENTS.md` baseline uses plain `N/A` (not template placeholder text), so fresh-adopt repos pass drift without manual editing

### Token Optimization Roadmap (Steps 1–7)

Completed full seven-step token payload optimization:

- Step 1 baseline: `~104,696` combined tokens
- Step 7 result: `~49,202` combined tokens (−53.0% overall; −36.2% L0+L1 strict comparable)
- Output-tier separation: Tier 1 (UI/label), Tier 2 (standard implementation), Tier 3 (architecture/audit)
- Task-level detection (`L0` / `L1` / `L2`) with domain gate short-circuit
- Domain summary loader with summary-first contract loading (KDC example: −38.7%)
- Payload audit logger with distinct `onboarding` audit lane

### Runtime Hardening

- Onboarding now has its own audit lane even when task level is `L1`
- Windows terminal output uses a safe replacement fallback for non-encodable Unicode
- `session_start.py` and `pre_task_check.py` threaded with task-level-aware tier output

### Authority and Rule Classification

- `governance/AUTHORITY.md` — authority table for all governance tools
- `governance/RULE_REGISTRY.md` — 14 rule packs with classification
- `governance_tools/authority_loader.py` — runtime authority resolution
- `governance_tools/rule_classifier.py` — repo-type-aware rule pack classification

### Test Coverage

- `1,386 tests` at v1.1.0 (up from `835` at v1.0.0-alpha)
- New test suites: `test_authority_loader`, `test_rule_classifier`, `test_task_level_detector`, `test_output_tier`, `test_payload_audit_logger`, `test_reasoning_compressor`, and more

## Known Limits

This release is still intentionally bounded:

- Workflow interception coverage is partial — git hook and CI gate adoption is improving but not fully closed
- Domain enforcement remains mixed/advisory-first in many places, not universally hard-stop
- Semantic verification is improving but still shallower than a full policy engine
- Rule classification per repo type is not yet differentiated at the checker level (same checks apply to service, tooling, library repos)
- External reviewer onboarding without author guidance is the next Beta gate milestone

## Recommended Evaluation Path

1. Install dependencies: `pip install -r requirements.txt`
2. Run quickstart smoke: `python governance_tools/quickstart_smoke.py --project-root . --plan PLAN.md --contract examples/usb-hub-contract/contract.yaml --format human`
3. Run trust signal overview: `python governance_tools/trust_signal_overview.py --project-root . --plan PLAN.md --release-version v1.1.0 --contract examples/usb-hub-contract/contract.yaml --format human`
4. Publish local status snapshot: `python governance_tools/trust_signal_snapshot.py --project-root . --plan PLAN.md --release-version v1.1.0 --contract examples/usb-hub-contract/contract.yaml --publish-docs-status --format human`
5. Read the publication summary: `python governance_tools/trust_signal_publication_reader.py --project-root . --docs-status --format human`
6. Publish release package: `python governance_tools/release_package_snapshot.py --version v1.1.0 --publish-docs-release --format human`
7. Read release package: `python governance_tools/release_package_reader.py --version v1.1.0 --project-root . --docs-release --format human`
8. Read release root: `python governance_tools/release_package_publication_reader.py --project-root . --docs-release-root --format human`
9. Run release surface overview: `python governance_tools/release_surface_overview.py --version v1.1.0 --format human`
10. Run phase gates: `bash scripts/verify_phase_gates.sh`
