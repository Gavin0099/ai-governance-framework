# v1.1.0

發版日期：2026-03-22

`v1.1.0` 是目前最後一個正式對外 release。它把 framework 從 `v1.0.0-alpha` 的早期 prototype，再往前推成一條可驗證的 external adoption path，並完成當時的 token-optimization roadmap。

注意：

- 這份文件描述的是 `v1.1.0` 發版當時的 release-facing 狀態
- `main` 分支後續已加入更多 runtime / closeout / advisory / status hardening
- 那些較新的能力不應回寫成 `v1.1.0` 當時已存在

## v1.1.0 主要新增

### 1. Cross-platform adoption toolchain

- `adopt_governance.py` 取代 `bash scripts/init-governance.sh`，成為 canonical adopt entrypoint
- Windows adoption path 正式成立
- `--refresh` 可重建 baseline hash 與 section inventory
- `--dry-run` 可先預覽變更
- framework root 支援 env 與 upward scan 自動發現

### 2. Governance drift 與 schema visibility

- drift checker 成為 16-check 檢查面
- fresh-adopt repo 的 baseline、placeholder、template-copy guard、inventory freshness 都進到同一條檢查路徑
- `minimum-legal-schema.md` 成為 onboarding reference
- `AGENTS.md` baseline 的 plain `N/A` 不再讓 adopt repo 因 template 未填而立即 drift

### 3. Token optimization roadmap

這版完成了當時規劃的 token payload optimization：

- Step 1 baseline：約 `104,696` tokens
- Step 7 result：約 `49,202` tokens
- output tier 分層：Tier 1 / Tier 2 / Tier 3
- `L0` / `L1` / `L2` task-level detection
- summary-first contract loading
- payload audit logger 與 onboarding audit lane

### 4. Runtime hardening

- onboarding 與 runtime validation 多了一條 audit lane
- Windows terminal 的 Unicode fallback 更穩定
- `session_start.py` 與 `pre_task_check.py` 已能輸出 task-level-aware tier output

### 5. Authority 與 rule classification

- `governance/AUTHORITY.md`
- `governance/RULE_REGISTRY.md`
- `governance_tools/authority_loader.py`
- `governance_tools/rule_classifier.py`

這讓 authority surface 與 rule pack classification 在 release 時已經開始收斂。

### 6. Test coverage

這版 release note 當時記錄的是：

- `1,386 tests` at `v1.1.0`
- 相較 `v1.0.0-alpha` 的 `835` tests 明顯成長

## v1.1.0 當時已知邊界

這版 release 仍是 bounded prototype，不是 fully closed policy engine：

- workflow interception coverage 仍然 partial
- domain enforcement 多數仍為 mixed / advisory-first，不是 universally hard-stop
- semantic verification 仍淺於 full policy engine
- external reviewer onboarding 雖然已進到 Beta gate，但還不是 everything-solved state

換句話說，`v1.1.0` 的價值不是宣稱 framework 已完整成熟，而是證明 adopt / runtime / release-facing surfaces 已形成可評估主線。
