## Runtime Injection Layer

- 新增 `docs/runtime-injection-layer-plan.md`
- 將這條線明確定義為四層：
  - Canonical Governance Source
  - Injection / Compilation
  - Observation
  - Enforcement
- 補上 `Observation` 的兩種型別：
  - consumption observation
  - execution observation
- 明確指出目前 repo 的缺口不是單純 schema，而是：
  - observation 尚未全面成為 enforcement 可依賴的 decision input
  - injection 尚未被抽象成真正的 compilation responsibility
- first slice 決定先做 architecture boundary + minimal slice，不先做 full compiler
