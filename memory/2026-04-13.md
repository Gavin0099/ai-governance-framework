## meiandraybook — Bug Fixes & Feature Repairs

### 1. 營銷推廣中心 404 修復
- **原因**：Next.js 15 的 `params` 是 Promise，需要 `await`，直接解構會導致 404
- **修復**：`src/app/admin/group-buys/[id]/promotion/page.tsx` — `params: Promise<{ id: string }>` + `await params`
- **規則**：Next.js 15 所有動態路由頁面的 `params` 都必須 await

### 2. Books Master 1000 本上限修復
- **原因**：Supabase 預設 limit 1000，DB 超過後頁面只顯示 1000
- **修復**：`src/app/admin/books/page.tsx` + `src/app/actions/group-buys.ts` → 加 `.limit(10000)`
- **狀態**：已 push，等 Cloudflare 部署確認

### 3. Bundle Import Null Title Bug（部分修復）
- **症狀**：新書 INSERT 失敗 — `null value in column "title"`，但 JSON 資料有效
- **診斷**：所有受影響書目的 title 在 JS 端都是有效字串，guard 沒有被觸發，問題在 Supabase 序列化層或 PostgREST
- **修復方向**：
  - 改為明確列出 INSERT 的每個欄位（不 spread sourceFields）
  - 批量 INSERT 失敗時 fallback 到逐本 INSERT，同時附上 `[title="..." isbn=...]` 診斷訊息
- **狀態**：已 push，等 Cloudflare 部署後重試觀察

### 4. 廣播訊息修復
- **原因 1**：`owner_id !== currentUserId` 限制，其他 admin（Gavin）無法廣播
- **原因 2**：逐個買家呼叫 `sendLinePushMessage` + `recordLinePushResult` = 2N subrequests → 超出 Cloudflare Workers 上限 → application error
- **修復**：
  - 權限改為 `verifyAdminCaller()` — 任何 admin 都可廣播
  - 改用 LINE Multicast API（`/v2/bot/message/multicast`）一次發給所有人，1 個 subrequest
  - Push log 由逐人記錄改為一次性寫入（`lineId: "multicast:N"`）
- **新增函數**：`sendLineMulticastMessage()` in `src/lib/line-notify.ts`
