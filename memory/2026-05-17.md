## Cross-Agent Temporal Governance (new direction identified)

Starting from session-based hook enforcement gap in Hearth, arrived at structural insight:

**behavioral constraint vs temporal obligation** are different governance types. When extended
to multi-agent (Claude + ChatGPT + Copilot):

- session lifecycle is agent-specific → governance anchor collapses
- repository state transition is cross-agent shared reality → stable anchor

Shift: `session_end → obligation` → `state_transition_Y → obligation_Z`

This is event-sourced governance. Implications:
- agent hook = optional producer, not authority source
- verification must be git-diff-based, not session-based
- trigger artifact != obligation artifact (must be separate to avoid self-referential loop)
